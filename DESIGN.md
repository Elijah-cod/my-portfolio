# Design System: Architectural Product Dossier

## North star

The site should feel like a precise product dossier marked up with cobalt architectural ink. It is bright, direct, and human, using Elijah's portrait as the main visual anchor and interface diagrams as evidence of systems thinking.

The three voice words are **architectural**, **tactile**, and **lucid**.

## Rules

- Use the light lavender paper as the primary canvas. Dark sections are reserved for narrative pivots and contact.
- Use cobalt as the committed brand color, coral as a physical counterweight, and mint only for small technical signals.
- Keep hero headings below `5.65rem`, section headings below `4rem`, and long-form text around `65-72ch`.
- Use Sora for all product-facing copy and Azeret Mono only for labels, dates, and system annotations.
- Prefer borders, visible grids, offset blocks, and CSS diagrams over repeated floating cards.
- Case studies must state the problem and architectural decision, not only list technologies.
- Motion is limited to purposeful entrance reveals and responsive navigation.
- Avoid glassmorphism, gradient text, oversized editorial serif headlines, generic developer dashboards, and identical three-card sections.
- Mobile layouts should preserve the narrative order and keep all primary actions reachable without horizontal scrolling.

## Signature components

- **Portrait field:** arch-shaped photograph with a coral offset shadow and small green index note.
- **Manifest strip:** cobalt discipline bar separating the hero from the supporting sections.
- **Principle rows:** numbered UI/UX principles with wide readable descriptions.
- **Case-study diagrams:** a different CSS-built system visualization for each selected project.
- **Article ledger:** border-led archive and readable long-form article layout with structured metadata.

# Impeccable Portfolio

Use this skill when working on this repository's portfolio UI.

## Goals

- Make the site feel authored, not generated.
- Keep the portfolio readable before making it dramatic.
- Let the layout use desktop width without turning headlines into posters.
- Show engineering depth through structure, motion, and information design.
- Preserve the AI systems and architecture narrative.

## Non-negotiables

- Avoid oversized display text that breaks reading flow.
- Avoid narrow left-only text columns with dead space on the right.
- Avoid generic dark-theme gradients with no hierarchy.
- Avoid icon-wall sections with no explanation.
- Avoid interactions that appear broken because there is no visible feedback.

## Visual direction

- Prefer one strong visual idea per section over many decorative effects.
- Use contrast in scale carefully: headers should lead, not overwhelm.
- Treat cards as part of a larger composition, not isolated boxes.
- Use spacing to group related information tightly and separate different ideas clearly.
- Keep the page feeling technical, but not sterile.

## Typography rules

- Hero headline should stay readable on a laptop without feeling like a billboard.
- Section titles should usually fit in 2-4 lines on desktop, not 6-8.
- Supporting copy should sit close enough to headings to read as one unit.
- Body text should stay comfortable for scanning and never feel cramped.
- If a title starts dominating a screenshot, reduce scale before changing everything else.

## Layout rules

- Use width intentionally: desktop sections should occupy the canvas with balance.
- Pair heading and supporting copy so they feel connected, not stranded.
- Keep cards aligned to a consistent rhythm across sections.
- Use asymmetry only when it improves emphasis or flow.
- Collapse to simpler stacked layouts on smaller screens.

## Interaction rules

- Buttons and demo actions must produce obvious feedback.
- Hover, focus, and reveal states should feel deliberate and consistent.
- Command palette, labs, and navigation should remain usable after visual changes.

## Workflow

1. Audit the current section for typography, measure, spacing, and hierarchy issues.
2. Decide the visual center of gravity for that section.
3. Reduce type scale before adding more layout complexity.
4. Rebalance the section using width, grouping, and alignment.
5. Check that the interaction still feels responsive and visible.
6. Re-read the section as a visitor, not as the implementer.

## Repository-specific guidance

- Keep the existing project links, portrait, logo, and portfolio narrative unless explicitly changed.
- Preserve the semantic search lab unless it is being replaced with something stronger.
- When editing the hero, optimize for trust, clarity, and identity before spectacle.
- When editing section intros, prefer concise titles over oversized editorial statements.
- When in doubt, choose clean, sharp, high-signal design over novelty.
